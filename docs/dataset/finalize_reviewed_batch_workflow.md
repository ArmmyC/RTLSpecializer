# Finalize a reviewed batch

Finalization connects the existing safe local stages after human review:

```text
prepare review batch -> manual review -> readiness check -> finalization smoke -> optional commit/publish decision
```

It does not review or edit answers, make draft rows ready, execute RTL, run EDA tools, call models, download data, or train anything. It only verifies that a batch already accepted by the readiness gate can pass promotion, deterministic release assembly, conservative baseline generation, and deterministic evaluation.

Run:

```bash
python scripts/dataset/finalize_reviewed_batch.py \
  --batch-dir data/review/verilog_eval_batch_001 \
  --processed-output data/processed/verilog_eval_validated_v0.1.jsonl \
  --promotion-report data/reports/verilog_eval_validated_v0.1_report.json \
  --release-name release_v0.1_plus_verilog_eval_001 \
  --release-output-dir data/releases \
  --candidate-output data/eval/candidates/rule_baseline_verilog_eval_001.jsonl \
  --eval-output-dir data/eval/runs/rule_baseline_verilog_eval_001 \
  --allow-source-overlap \
  --json
```

The command first writes `readiness_report.json` and `readiness_report.md` under the batch directory. Unless `all_rows_ready` is true, it stops before promotion and records the reason in `finalization_summary.json` and `finalization_summary.md`.

When readiness passes, promotion uses target status `validated`, rejects stub answers, and fails on any rejected row. The promoted file is strictly validated before release assembly. The release combines the golden dataset with the promoted rows using seed 7 by default, then the test split receives conservative rule-baseline candidates and deterministic evaluation.

Every configured output is protected by default. Use `--force` only when intentionally replacing outputs generated by this command. Cleanup is limited to the exact report/files and the named release/evaluation directories; inputs and parent directories are never removed. Outputs inside `.local_data` are rejected.

Generated outputs remain local by default. Finalization does not scientifically certify RTL correctness, licenses, provenance, or dataset quality and does not replace human review. Inspect the promotion report, release manifest and dataset card, evaluation metrics, provenance, and licenses before intentionally publishing or committing any approved artifact.
