# Fix Spec: Finalization Output Safety Hardening v0.1

## 1. Goal

Harden `scripts/dataset/finalize_reviewed_batch.py` output cleanup and preflight behavior before it is used on real local review batches.

The current finalization implementation is broadly on-spec and successfully connects readiness checking, strict promotion, release assembly, baseline candidate generation, evaluation, and summary reports. However, its `--force` cleanup path needs stronger path-safety guarantees because generated directory cleanup currently resolves paths before deletion. A directory symlink could therefore cause deletion outside the intended managed output path.

This fix spec is focused and should not change the pipeline semantics.

## 2. Non-goals

Do not build:

- model training,
- model inference,
- LLM calls,
- EDA execution,
- RTL simulation/synthesis/equivalence/toggle/power analysis,
- schema changes,
- automatic review,
- automatic answer editing,
- external downloads,
- publishing or committing generated artifacts.

Do not rewrite the finalization workflow unless needed for path-safety.

## 3. Required fixes

### FR-1: Never follow symlinks during managed output cleanup

The current `_remove_dir` uses:

```python
shutil.rmtree(path.resolve())
```

This must be changed.

Requirements:

- If a managed directory output path is a symlink, fail safely instead of following it.
- Never call `shutil.rmtree()` on `path.resolve()`.
- Delete only the exact path that was explicitly configured as a managed output directory.
- For directories, use `shutil.rmtree(path)` only after verifying:
  - `path.exists()` or `path.is_symlink()`,
  - `not path.is_symlink()`,
  - `path.is_dir()`,
  - path is not inside `.local_data`,
  - path is not an input path,
  - path is not a parent of an input path.
- For files, remove only regular files or file symlinks at the exact managed file path using `path.unlink()`. Do not follow file symlinks to delete targets.

### FR-2: Preflight must reject managed output directory symlinks

During preflight, if any managed directory output path exists as a symlink, reject it even if it points to a directory.

Managed directory outputs are:

```text
release_dir
eval_dir
```

The error should be clear, for example:

```text
managed output directory must not be a symlink: <path>
```

### FR-3: Preflight must reject dangerous managed output ancestry

Add explicit checks that managed output directories are not:

- the filesystem root,
- the repository root,
- the user's home directory,
- a parent of `batch_dir`, `selected_rows.jsonl`, `reviewed_rows.jsonl`, or `golden_input`,
- inside `.local_data`,
- overlapping each other.

The existing code already handles some overlap cases. Keep those checks and strengthen them where needed.

Do not overfit to a specific absolute repository path; derive repository root from the script path when practical.

### FR-4: Summary/report overwrite UX

Finalization writes these reports under the batch directory:

```text
readiness_report.json
readiness_report.md
finalization_summary.json
finalization_summary.md
```

These are generated by the finalization command itself.

Behavior should be:

- Without `--force`, fail if these files already exist, preserving current output protection behavior.
- With `--force`, replace only these exact report files.
- Unknown files in the batch directory, such as `reviewer_notes.md`, must be preserved.

This behavior already mostly exists; add tests and clarify docs.

### FR-5: Preflight errors should still be visible in CLI JSON/text

If preflight fails before reports can be written, the CLI should still return a clear JSON/text summary with:

```json
{
  "ok": false,
  "errors": [...]
}
```

Do not require writing summary files when the batch directory is missing or unsafe. It is acceptable not to write reports when preflight fails before output paths are safe.

### FR-6: Add tests

Update:

```text
tests/dataset/test_finalize_reviewed_batch.py
```

Add tests for:

1. `--force` rejects a symlinked `release_dir` and does not delete the symlink target.
2. `--force` rejects a symlinked `eval_dir` and does not delete the symlink target.
3. `--force` removes only exact generated report files in `batch_dir`, preserving unknown files.
4. `--force` removes only exact managed pipeline outputs, preserving unknown files in parent directories.
5. Managed directory output cannot be repository root, home directory, or filesystem root.
6. Managed output directory cannot be a parent of the batch directory or selected/reviewed input files.
7. CLI JSON output is parseable when preflight fails.

Use temporary directories only. Do not add real VerilogEval content.

### FR-7: Docs

Update:

```text
docs/dataset/finalize_reviewed_batch_workflow.md
```

Clarify:

- `--force` replaces only exact managed outputs created by finalization.
- Symlinked managed output directories are rejected.
- Parent directories, input files, `.local_data`, and unknown files are not deleted.
- If preflight fails, no promotion/release/evaluation occurs.

## 6. Definition of done

Done only when:

- `--force` cleanup never follows directory symlinks.
- Symlinked managed output directories are rejected.
- Managed output directories cannot target root, home, repo root, input parents, `.local_data`, or overlapping output trees.
- Unknown files in batch/output parent directories are preserved.
- Preflight failure produces parseable CLI JSON.
- Existing finalization success/failure tests still pass.
- No model calls, EDA calls, downloads, training, schema changes, automatic review, or automatic commits are introduced.

## 7. Codex implementation instructions

Implement this fix spec exactly.

Keep the finalization workflow behavior the same, but make output cleanup and `--force` path handling safer.

After finishing, commit and push. Summarize changed files, test commands, smoke commands, results, and tradeoffs.
